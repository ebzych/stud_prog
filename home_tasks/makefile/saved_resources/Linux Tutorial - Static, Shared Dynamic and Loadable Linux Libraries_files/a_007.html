    Actually, I think this article needs a lot of polishing as it's hard to make out what the author is trying to say.
    Let's just take the section Linux Library Types.
    The author uses "statically aware". This is somewhat cryptic. Please be more explicit.
    Then there is the half-sentence
    Thus the following link command gcc src-file.c -lm -lpthread
    The word "Thus" is somewhat inappropriate here. I would have put "Consider" which nods the reader to read further on, whereas "Thus" suggests that a conclusion has been reached. Oh, ok, if I continue reading I do workout what was intended, but the reader has to work around what was written.
    Then there is the statement:
    Libraries are typically names with the prefix "lib"
    well, no. Libraries *are* not names. They *have* names.

        In the section for: Dynamically Linked "Shared Object" Libraries: (.so)

        In the subsection:
        Library creation example:
        gcc -Wall -fPIC -c *.c
        gcc -shared -Wl,-soname,libctest.so.1 -o libctest.so.1.0 *.o

        Assuming prog.c, ctest1.c and ctest2.c are all in the same directory, The above commands will unnecessarily place an object module prog.o in the shared library. Instead of *.c and *.o in these commands, I would recommend changing them to ctest*.c and ctest*.o
